# Ceremonies — Pulse 360°

Ceremonies are structured team alignments. Auto-triggered ceremonies run when their condition matches; manual ceremonies run only when Russ asks.

## Definitions

### design-review
- **Trigger:** manual ("design meeting", "let's plan X")
- **When:** before
- **Facilitator:** Miller
- **Participants:** Miller, plus the domain owners for the feature (Naomi / Amos / Alex as relevant)
- **Purpose:** Align on approach, surface dependencies, agree on the work breakdown before building.

### feature-kickoff
- **Trigger:** auto
- **When:** before
- **Condition:** a multi-domain feature touching 3+ agent domains
- **Facilitator:** Miller
- **Participants:** all domain owners for the feature
- **Purpose:** Decompose, assign, and identify hard data dependencies before fan-out.

### pre-ship
- **Trigger:** auto
- **When:** after
- **Condition:** a user-facing artifact or content change is finalized
- **Facilitator:** Miller
- **Participants:** Bobbie (quality), Rai (RAI/content safety)
- **Purpose:** Final quality + responsible-AI pass before merge/ship.

### retro
- **Trigger:** manual ("run a retro")
- **When:** after
- **Facilitator:** Miller
- **Participants:** full team
- **Purpose:** What went well, what to improve, capture learnings as skills/decisions.
