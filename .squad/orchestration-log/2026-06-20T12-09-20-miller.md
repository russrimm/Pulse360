### Critical Sequencing Decisions

1. **M0 independence:** Sanitization ships as a complete, reviewable unit before any parsing/normalization work. Resolves Rai's 🔴 Critical XSS mandate without blocking larger refactor.
2. **Image proxy parallel:** WI-02 (Amos) runs alongside WI-01 (Alex); DOMPurify hook in sanitize.ts auto-rewrites img[src] to /api/image-proxy?url=... on every sanitization call.
3. **Parser parallelization:** WI-07/08/09 (RSS/JSON/HTML parsers) are independent; route migration (WI-10) gates on RSS + HTML-table parsers only, allowing Naomi's JSON parser to follow independently.
4. **Error envelope isolation:** WI-14 (Amos) is separate from WI-10 (route parsing) — different concerns (format vs. error semantics), focused PRs, simpler rollback.
5. **Cache comments:** WI-15 mandates code comments for intentional cache-behavior changes (M365 revalidate 1800s, Fabric 3600s) per Alex's original flagging.
6. **MessageDetail ordering:** WI-04 acceptance criteria require sanitization of processedContent (after regex transform), preserving existing behavior while closing XSS gap.
